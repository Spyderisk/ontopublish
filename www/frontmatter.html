Spyderisk Ontopublish

Welcome to the Spyderisk ontology publisher. The directory structure is as follows:

/ns
Vocabularies and ontologies
/sh
Shape expressions (currently ShEx), imposing constraints
/v
Versioning/provenance logs (semantics described here)
/p
Instance data, including examples

Generally speaking, an RDF/S vocabulary will have a corresponding shape expression file (used to impose constraints on instance data), as well as a version log. For example, the Spyderisk V1 core vocabulary is found at /ns/core, and it has a corresponding shape expression document /sh/core, as well as a version log /v/core.

BEAM Ontology: ontology of the Erlang virtual machine

http://ontology.spyderisk.org/ns/beam

The BEAM is Erlang's virtual machine. This is a lightweight, demonstration ontology with fairly simple semantics based on the RDF schema (RDF/S) alone, i.e. no dependency on OWL. It models what the BEAM is doing at a high level, and some of OTP (many of the bits which make Erlang Erlang).

The aim is to use it to demonstrate versioning an ontology with respect to what RDF/S alone gives us.
